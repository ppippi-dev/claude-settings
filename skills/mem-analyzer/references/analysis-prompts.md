# 패턴 분석 가이드

observations 데이터에서 추천을 도출하기 위한 분석 기준.

## 스킬 추천 기준

다음 패턴이 발견되면 스킬 추천:

| 패턴 | 스킬 추천 조건 |
|------|---------------|
| 동일 워크플로우 반복 | 3회 이상 유사한 작업 순서가 반복 |
| 특정 파일 형식 처리 | 같은 확장자 파일을 반복 처리 |
| 외부 도구 연동 | 특정 CLI/API를 반복 호출 |
| 코드 생성 패턴 | 비슷한 구조의 코드를 반복 생성 |

스킬 제안 시 포함할 내용:
- 스킬명 (kebab-case)
- 트리거 조건
- 주요 워크플로우 단계
- 필요한 리소스 (scripts, references, assets)

## 훅 추천 기준

다음 패턴이 발견되면 훅 추천:

| 패턴 | 훅 유형 |
|------|--------|
| 세션 시작 시 항상 같은 작업 | PreToolUse 또는 SessionStart |
| 특정 도구 사용 전/후 항상 같은 작업 | PreToolUse / PostToolUse |
| 커밋/푸시 전 항상 같은 검증 | PreToolUse (git) |
| 파일 저장 후 항상 같은 작업 | PostToolUse (Write/Edit) |

훅 제안 시 포함할 내용:
- 훅 유형 (PreToolUse, PostToolUse, etc.)
- matcher 조건
- 실행할 명령어
- timeout 설정

## settings.json 추천 기준

다음 패턴이 발견되면 settings.json 추천:

| 패턴 | 설정 항목 |
|------|----------|
| 특정 명령어 반복 승인 | allowedTools / allowPrompts |
| 환경 변수 반복 설정 | env |
| MCP 서버 관련 작업 | mcpServers |

## CLAUDE.md 추천 기준

다음 패턴이 발견되면 CLAUDE.md 추천:

| 패턴 | 규칙 유형 |
|------|----------|
| 반복되는 의사결정 | 의사결정 기준 규칙 |
| 코딩 컨벤션 관련 지적 | 코드 스타일 규칙 |
| 특정 도구/라이브러리 선호 | 도구 사용 규칙 |
| 워크플로우 순서 강조 | 작업 순서 규칙 |

## 분석 우선순위

1. **빈도 기반**: 자주 반복되는 패턴 우선
2. **최신성 기반**: 최근 세션의 패턴에 가중치
3. **영향도 기반**: 여러 세션에 걸쳐 나타나는 패턴 우선
