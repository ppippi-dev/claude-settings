# History

세션의 작업 내역을 ~/.claude/history/YYYY-MM-DD/<title>.md에 기록한다.

## 워크플로우

1. 현재 세션 분석
2. 히스토리 파일 생성
3. 완료 알림

## 1. 세션 분석

현재까지의 대화에서 다음을 추출:

- **요청 요약**: 사용자가 요청한 핵심 내용 (1-2문장)
- **Claude의 질문**: AskUserQuestion으로 물어본 내용과 사용자 답변
- **의사결정**: 중요한 기술적/설계적 결정 사항
- **사용 도구**: 주요 도구 사용 내역 (Read, Edit, Bash 등)
- **결과물**: 생성/수정된 파일 목록 및 주요 변경 사항

## 2. 히스토리 파일 생성

### 경로 규칙

```text
~/.claude/history/YYYY-MM-DD/<title>.md
```

- `YYYY-MM-DD`: 오늘 날짜
- `<title>`: 작업 내용 기반 자동 생성 (영문 kebab-case, 3-5단어)
  - 예: `create-history-skill`, `fix-auth-bug`, `add-dark-mode`

### 파일 포맷

```markdown
# <Title in Korean>

- **날짜**: YYYY-MM-DD HH:MM
- **작업 시간**: 약 N분

## 요청 요약

[사용자의 핵심 요청 1-2문장]

## 질문과 답변

| 질문 | 답변 |
|------|------|
| [Claude가 물어본 질문] | [사용자 답변] |

## 의사결정

- [결정 1]: [이유]
- [결정 2]: [이유]

## 주요 작업

1. [작업 1]
2. [작업 2]

## 결과물

### 생성된 파일
- `path/to/file.ext` - 설명

### 수정된 파일
- `path/to/file.ext` - 변경 내용 요약

## 메모

[추가 참고사항, 다음 단계, 미해결 이슈 등]
```

## 3. 완료 처리

1. 디렉토리 없으면 생성: `mkdir -p ~/.claude/history/YYYY-MM-DD`
2. 히스토리 파일 작성
3. 저장 경로 알림
